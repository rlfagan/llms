## 8. Security Concerns

### CVEs
**No CVEs** were explicitly referenced in the codebase. Both scan reports confirm `0 active CVEs (0 total)`.

### Identified Security Issues

| Concern | Severity | Details |
|---|---|---|
| **API key in CI environment** | Medium | `ANTHROPIC_API_KEY` is passed as a GitHub secret to the workflow. Proper secret management is in place, but the `scan.py` script could potentially leak the key in logs if not carefully written. |
| **Unrestricted model weight download** | Medium | `download_top10.py` can download full model weights (`DOWNLOAD_WEIGHTS=1`) from Hugging Face without integrity verification (no hash/checksum validation). This is a supply-chain risk. |
| **No model provenance verification** | Medium | Model metadata is fetched over HTTPS from Hugging Face API but there is no signature verification or hash pinning. A compromised or poisoned model could be ingested. |
| **Broad write permissions** | Low | The GitHub Actions workflow has `contents: write` and `pull-requests: write` permissions. An auto-commit step (`git push`) runs on non-PR events. |
| **NSFW detection model presence** | Informational | `Falconsai/nsfw_image_detection` is catalogued, which processes sensitive content. If deployed, ensure appropriate data handling and privacy controls. |
| **EU AI Act / GDPR annotations** | Informational | The AI-BOM flags multiple models as `eu-ai-act-risk: high` and `gdpr-relevant: true`. Organizations using these models should ensure regulatory compliance. |
| **No `scan.py` available for review** | Low | The core scanning logic is not included in the provided files, making it impossible to audit prompt injection risks or data leakage in the Claude API calls. |

## Summary

This repository is an **AI model catalogue and automated scanning platform**. It does not run ML inference directly but instead:

1. **Catalogues metadata** for the top 10 most-downloaded Hugging Face models (sentence transformers, BERT, CLIP, ELECTRA, XLM-RoBERTa, CLAP, and a ViT-based NSFW detector)
2. **Runs an Anthropic Claude-powered CI/CD pipeline** (`scan.py` via GitHub Actions) that automatically scans the repository for AI components and generates rich HTML reports, CycloneDX 1.7 AI-BOMs, and interactive topology diagrams
3. **Tracks additional models** via `aibom.py` (not provided) including LlamaIndex, LLaMA, Mistral/Mixtral, and others

**Key statistics:**
- **10 Hugging Face models** catalogued with metadata (no weights stored by default)
- **12–20 models** detected by the automated scanner (including references in `aibom.py`)
- **0 CVEs** found in the codebase
- **1 LLM integration** (Anthropic Claude for scanning)
- **Primary frameworks:** Transformers, Sentence-Transformers, PyTorch, TensorFlow, JAX, ONNX

**Primary risks** center on supply-chain integrity (no model hash verification), the CI secret handling, and regulatory compliance for the high-risk-flagged models.
