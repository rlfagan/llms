Scanner's Own Orchestration Pipeline

The scanner itself implements an AI-powered multi-stage pipeline in scan.py:

1. File Collection — Walks repository tree, filters by AI-relevance keywords and file types

2. Model Reference Extraction — Regex-based detection of model names across source files

3. Claude API Analysis — Sends collected code to Anthropic Claude for structured AI component analysis

4. AI-BOM Generation — aibom.py produces CycloneDX 1.7-compliant AI Bill of Materials

5. Topology Generation — topology.py builds interactive graph visualizations (Cytoscape.js)

6. Report Assembly — Produces Markdown, HTML, and JSON (CycloneDX BOM) outputs

6. Prompt Engineering

System Prompt Architecture (scan.py)

The scanner constructs structured prompts for Claude that include:

  • AI keyword dictionary (AI_KEYWORDS): 35+ terms used to filter and prioritize relevant files
  • Section-based output format: The prompt directs Claude to produce analysis under the exact SECTIONS headings defined in scan.py
  • Model reference context: Detected model references and their source locations are passed to Claude as enrichment data

Topology Prompt (topology.py)

A separate prompt (build_topology_prompt) is used to generate structured JSON describing the AI system topology, which is then rendered as an interactive graph.

8. Security Concerns

API Key Exposure Risk

ConcernSeverityDetails
Anthropic API key in CIMediumANTHROPIC_API_KEY is passed via GitHub Secrets — properly handled, but any workflow modification could expose it in logs
No key validationLowscan.py imports anthropic directly; no visible error handling for missing/invalid API keys

Scanner Output Filtering Gaps

ConcernSeverityDetails
Stale scanner report ingestionMediumSKIP_SCANNER_PATTERNS regex attempts to exclude prior scan outputs (Grype, Trivy, Snyk, etc.) but complex nested paths could bypass this
MAX_FILE_SIZE / MAX_TOTAL_CHARS limitsLowFiles >100KB and total content >180KB are truncated, potentially missing security-relevant code in large files

Supply Chain Risks

ConcernSeverityDetails
Model weight downloadsHighmodels/download_top10.py can download full model weights from Hugging Face Hub when DOWNLOAD_WEIGHTS=1 — no integrity verification (hash checking) is performed
No dependency pinningMediumpip install anthropic requests in CI uses unpinned versions, vulnerable to dependency confusion or supply chain attacks
Unvalidated HF API responsesLowdownload_model_metadata() uses urllib.request.urlopen() with no certificate pinning or response validation

AI-Specific Concerns

ConcernSeverityDetails
Prompt injection via scanned reposHighScanned repository file contents are sent directly to Claude — a malicious repository could contain prompt injection payloads in source files that alter scan output
AI-BOM false positivesMediumThe BOM in reports/ shows llama-index, llama_index, llama, mixtral, and mistral-ai detected as models when they are actually just regex pattern strings or library registry entries in aibom.py — not active model usage
No output validationMediumClaude's analysis output is written directly to reports without sanitization — could include misleading or hallucinated security findings

CVE Detection

The scanner includes a CVE pattern matcher (CVE_PATTERN = re.compile(r'CVE-\d{4}-\d{4,7}')) and a classifier to distinguish active vulnerabilities from changelogs/fixes. No specific CVE IDs were found referenced in this codebase.
