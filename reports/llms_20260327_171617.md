### Scanner's Own Orchestration Pipeline

The scanner itself implements an AI-powered multi-stage pipeline in `scan.py`:

1. **File Collection** — Walks repository tree, filters by AI-relevance keywords and file types
2. **Model Reference Extraction** — Regex-based detection of model names across source files
3. **Claude API Analysis** — Sends collected code to Anthropic Claude for structured AI component analysis
4. **AI-BOM Generation** — `aibom.py` produces CycloneDX 1.7-compliant AI Bill of Materials
5. **Topology Generation** — `topology.py` builds interactive graph visualizations (Cytoscape.js)
6. **Report Assembly** — Produces Markdown, HTML, and JSON (CycloneDX BOM) outputs

## 6. Prompt Engineering

### System Prompt Architecture (`scan.py`)

The scanner constructs structured prompts for Claude that include:

- **AI keyword dictionary** (`AI_KEYWORDS`): 35+ terms used to filter and prioritize relevant files
- **Section-based output format**: The prompt directs Claude to produce analysis under the exact `SECTIONS` headings defined in `scan.py`
- **Model reference context**: Detected model references and their source locations are passed to Claude as enrichment data

### Topology Prompt (`topology.py`)

A separate prompt (`build_topology_prompt`) is used to generate structured JSON describing the AI system topology, which is then rendered as an interactive graph.

## 8. Security Concerns

### API Key Exposure Risk

| Concern | Severity | Details |
|---------|----------|---------|
| **Anthropic API key in CI** | Medium | `ANTHROPIC_API_KEY` is passed via GitHub Secrets — properly handled, but any workflow modification could expose it in logs |
| **No key validation** | Low | `scan.py` imports `anthropic` directly; no visible error handling for missing/invalid API keys |

### Scanner Output Filtering Gaps

| Concern | Severity | Details |
|---------|----------|---------|
| **Stale scanner report ingestion** | Medium | `SKIP_SCANNER_PATTERNS` regex attempts to exclude prior scan outputs (Grype, Trivy, Snyk, etc.) but complex nested paths could bypass this |
| **MAX_FILE_SIZE / MAX_TOTAL_CHARS limits** | Low | Files >100KB and total content >180KB are truncated, potentially missing security-relevant code in large files |

### Supply Chain Risks

| Concern | Severity | Details |
|---------|----------|---------|
| **Model weight downloads** | High | `models/download_top10.py` can download full model weights from Hugging Face Hub when `DOWNLOAD_WEIGHTS=1` — no integrity verification (hash checking) is performed |
| **No dependency pinning** | Medium | `pip install anthropic requests` in CI uses unpinned versions, vulnerable to dependency confusion or supply chain attacks |
| **Unvalidated HF API responses** | Low | `download_model_metadata()` uses `urllib.request.urlopen()` with no certificate pinning or response validation |

### AI-Specific Concerns

| Concern | Severity | Details |
|---------|----------|---------|
| **Prompt injection via scanned repos** | High | Scanned repository file contents are sent directly to Claude — a malicious repository could contain prompt injection payloads in source files that alter scan output |
| **AI-BOM false positives** | Medium | The BOM in `reports/` shows `llama-index`, `llama_index`, `llama`, `mixtral`, and `mistral-ai` detected as models when they are actually just regex pattern strings or library registry entries in `aibom.py` — not active model usage |
| **No output validation** | Medium | Claude's analysis output is written directly to reports without sanitization — could include misleading or hallucinated security findings |

### CVE Detection

The scanner includes a CVE pattern matcher (`CVE_PATTERN = re.compile(r'CVE-\d{4}-\d{4,7}')`) and a classifier to distinguish active vulnerabilities from changelogs/fixes. **No specific CVE IDs were found referenced in this codebase.**
