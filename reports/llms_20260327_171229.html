6. Prompt Engineering

System Prompt in scan.py

The scanner constructs a structured prompt for Claude that:

  • Provides all collected repository file contents as context
  • Requests analysis organized into specific sections matching SECTIONS constant
  • Instructs Claude to identify AI frameworks, LLM integrations, embeddings, agents, models, prompts, infrastructure, and security concerns
  • Expects markdown-formatted structured output with section headings

Model Reference Extraction

The MODEL_PATTERNS list in scan.py serves as a prompt-adjacent heuristic to pre-identify model references before sending to Claude, supplementing the LLM analysis with deterministic pattern matching.

AI Keyword Detection

The AI_KEYWORDS set acts as a relevance filter to determine which files to include in the Claude analysis prompt, effectively functioning as a retrieval heuristic.

8. Security Concerns

CVE Detection Capability

The scanner includes a CVE_PATTERN regex (CVE-\d{4}-\d{4,7}) to detect CVE references in scanned repositories. It also includes a CVE context classifier that distinguishes between:

  • CVEs in changelogs/advisories (fix references)
  • CVEs in dependency manifests (active vulnerable dependencies)

No specific CVE IDs are hardcoded or mentioned in the codebase.

Identified Security Concerns

ConcernSeverityDetails
API Key in CI/CDMediumANTHROPIC_API_KEY stored as GitHub Actions secret and passed as environment variable. Proper secret management, but key exposure risk in logs if not careful.
Unbounded file readingLowscan.py reads files up to 100KB (MAX_FILE_SIZE) and total 180KB (MAX_TOTAL_CHARS), but errors="replace" could mask malicious content.
Auto-commit to repoMediumThe workflow auto-commits reports back to the repo with [skip ci], which could be exploited if scan output contains injected content from a malicious repository.
PR comment injectionMediumScan results posted as PR comments (up to 65KB) — if a scanned repo contains crafted content, it could inject markdown/links into PR comments.
No input validation on repo URLMediumThe scanner appears to accept repository URLs without strict validation, potentially enabling SSRF or path traversal if misused.
Model weight downloadsHighdownload_top10.py can download full model weights from Hugging Face Hub (DOWNLOAD_WEIGHTS=1). No integrity verification (hash checking) is performed on downloaded weights, risking supply chain attacks via poisoned models.
Scanner output self-exclusionLowSKIP_SCANNER_PATTERNS attempts to avoid ingesting stale scan results, but the pattern list may not be comprehensive enough to prevent confusion attacks.
No Claude response validationMediumThe scanner parses Claude's response as structured content but the truncated code doesn't show robust validation of Claude's output, risking prompt injection from malicious repository content being reflected in reports.
